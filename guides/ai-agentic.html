הבסיס

הבהרה קריטית: אלו לא אותן השכבות

העולם מתחלק כיום לשלוש רמות עבודה. חשוב להבין איפה אתם נמצאים כדי לא "לירות בצרעה עם תותח":

🧠

המוח (The Brain)

צ'אטים כמו Claude, ChatGPT, Gemini. הם חושבים איתך, מתכננים וכותבים קטעי קוד, אבל לא "נוגעים" בקבצים שלך. אבל, אמנם בגרסת הצ'אט הפשוטה והראשונית שכולנו מכירים הם לא ניגשים לקבצים שלכם במחשב או באפליקציה, ישנן יכולות שכן מאפשרות להם לעשות דברים שקצת מטשטשים את הגבולות באמצעות אינטגרציות עם כלים צד ג' שכן משפיעים על הקבצים המקומיים. Codex לדוגמה, Google AI Studio - יכולים לדבר עם ריפוזיטורי וליצור ולשנות קבצי פרויקט פיתוח. זו גם המטרה שלהם - להיות עוזר פיתוח. לכל אחד יש את ה"קטע" שלו שמייחד אותו.

🛠️

הידיים (The Agents)

כלים כמו Claude Code. הם סוכנים שרצים בתוך המערכת שלך ומבצעים פקודות בשמך. מדובר בשכבה שיושבת "מעל" - של סוכנים שיכולים לבצע פקודות בשמך. המשתמש עדיין "מדבר" בשפה שלו. אבל היכולות של הסוכן שונות מאלה של הצ'אטים.

🏠

הבית (Agentic IDE)

סביבות כמו Cursor ו-Antigravity. כאן הכל קורה במקום אחד - הקוד, הצ'אט, והסוכן שמריץ הכל. זו הפלטפורמה, המעטפת, שמאפשרת לכם גם לדבר עם הסוכנים וגם לגעת בקוד, אם תרצו. היכולות של הפלטפורמה מאפשרות לה ליצור הכל מאפס, לצרכי פיתוח. החל משלב התכנון, הלאה לסקיצה, מחקרי שוק, אפילו תמונות, צילומי מסך, בדיקות QA ובסוף גם להביא את זה לסביבת Production בזכות אינטגרציות רבות שיש לפלטפורמות הללו. ל-Antigravity גישה למודלים מתקדמים מאוד והיא יחסית נרחבת לעומת הנהוג בשוק. לכן, ניתן למקסם את יכולות המודלים עצמם.

צלילה לעומק

ארבעת הכלים - מה באמת קורה בפנים

Layer 1 - The Brains ללא התקנה

1. Web Chats: Claude, ChatGPT, Gemini

אלו הפלטפורמות שכולם מכירים. נגישות מהדפדפן, ללא התקנה.

  • ChatGPT (OpenAI): הוותיק. חזק בלוגיקה ובאינטגרציות עם כלים כמו Canva, GitHub ועוד.
  • Claude (Anthropic): מצטיין בכתיבה ובדיוק גבוה בהוראות ובקוד.
  • Gemini (Google): חזק מאוד בטיפול בכמויות ענקיות של מידע - חלון קונטקסט גדול במיוחד.
💡 מתי להשתמש?
  • - כשצריך לחשוב על רעיון, לכתוב טקסטים, לשאול "איך עושים X" - בלי להוריד שום דבר.
  • - מוקאפים ופרוטוטייפים: "תבנה לי אפליקציה אינטראקטיבית" - כדי לראות את הרעיון תופס חיים. הכלים האלה מחזירים קבצים שאפשר ממש לקחת, להטמיע ולהפוך לאתר או דף נחיתה בצורה פשוטה.
  • - מחקרי שוק: סקירת מתחרים, ניתוח טקסטים, סיכום מסמכים.
  • - ניתוח ותכנון: קבלת החלטות, ניסוח, הכנת מצגות.
  • - ועוד - ממשיכים לגלות כל הזמן.
Layer 2 - CLI Agent

2. Claude Code - הסוכן בשורת הפקודה

Claude Code הוא כלי של Anthropic שרץ ישירות בטרמינל שלך. הוא לא IDE - הוא סוכן שמדבר עם הקוד שלך ישירות, מבצע שינויים, מריץ פקודות, וכותב בדיקות.

  • מודלים: מבוסס על מודלי Claude של Anthropic.
  • מתאים למתחילים? לא מומלץ ללא רקע טכני. דורש היכרות עם הטרמינל ושורת הפקודה. אם הטרמינל לא מוכר לך - כדאי להתחיל עם Cursor.
  • לאילו משימות מצטיין: תיקון באגים בפרויקט קיים, Refactoring, כתיבת בדיקות אוטומטיות, ביצוע שינויים גדולים בקוד בפקודה אחת.
⚠️ שורה תחתונה: כלי עוצמתי מאוד - אבל בידיים של מי שמבין קוד. למי שמגיע בלי רקע - יש כלים ידידותיים יותר.

3. Cursor

ה-IDE הפופולרי ביותר כיום. הוא נראה ומרגיש כמו VS Code (העורך המקובל בעולם) אבל עם "סוכן" מובנה בשם Composer שיכול לכתוב פרויקטים שלמים מאפס.

יתרון מרכזי:

חוויית משתמש נוחה וחלקה, מתקדמת. צבר כבר זמן בשוק ומשתמשים רבים. פופולארי ומבוקש ושינה את חוקי המשחק כי הוא השילוב המושלם לצרכים רבים וקהל רחב שלא פגש קוד לקחת רעיון ולצאת עם - בום! ויש גם פומו, כמו תמיד .

4. Google Antigravity

התשובה של גוגל ל-Cursor. הוא מתמקד בניהול סוכנים מרובים על פני כמה פרויקטים (Workspaces) במקביל, עם יכולת לשלוט בדפדפן כדי לבדוק את האתר שהסוכן בנה.

יתרון מרכזי:

אורקסטרציה חזקה ושליטה בדפדפן "כמו בן אדם". מדובר בכלי מאוד דומה למתחרה שלו. גוגל שחררו אותו כ"תשובה" למתחרה ובינתיים, הוא חינמי, עדיין נחשב לבטא. בעתיד ייתכן וייגבה עבורו תשלום

⚠️ חשוב להבין שניהם - הם סביבת העבודה. הארגז. יש מגרות, קומות, אפשר להזיז ולשחק. יכלי העבודה בפועל - המברגים השונים נקרא לזה- אלה המודלים. אם כך, שתי הפלטפורמות מאפשר גישה למודלים. המשתמש יכול לבחור ברמת הצט אם הוא רוצה לעבוד עם מודל אחד ובצט אחד להחליף מודל וגם באמצע הצט. זה מאפשר גמישות רבה. אבל כמות הקרדיטים החינמיים שונה ממודל למודל ובין הפלטפורמות עצמן. באנטיגרביטי לדוגמא מודל Flash כנראה יספיק לנצח ומתחדש כל 4 שעות. יתר המודלים החזקים של שאר המוחות מתומחרים אחרת ולמרות זאת, אפשר להקים ולפתח המון עם מה שמקבלים, כולל עם Sonnet. מחירים ניתן לראות באתרים בדיוק כמה כל מודל עולה וכמובן ניתן לרכוש עוד קרדיטים לשימוש בתוך הפלטפורמה.

מדריך מעשי

איך מתחילים - מדריך שלב-שלב

🖥️ Cursor - הכי קל להתחיל

שלב 1: הורדה

נכנסים ל-cursor.com, לוחצים Download. קובץ התקנה רגיל. עובד על Mac, Windows, Linux.

שלב 2: פתיחת פרויקט

פותחים את Cursor, יוצרים תיקייה חדשה (New Folder) - זה יהיה הפרויקט שלכם.

שלב 3: פתיחת הסוכן (Composer)

לוחצים Ctrl+I (Windows) או Cmd+I (Mac). מופיע חלון הצ'אט של הסוכן.

שלב 4: הבקשה הראשונה

כותבים בעברית או באנגלית, לדוגמה: "Build me a modern landing page for a yoga studio with a booking form" - הסוכן מתחיל לכתוב קבצים.

שלב 5: העלאה לאוויר

מחברים חשבון Vercel (חינם) ולוחצים Deploy - האתר עולה עם דומיין זמני.

⏱️ זמן מוערך: 15-25 דקות מהורדה ועד אתר שרץ באינטרנט.

⌨️ Claude Code - למשתמשים טכניים

⚠️ לא מומלץ למתחילים ללא רקע. אם הטרמינל לא מוכר לך - תתחיל עם Cursor.
דרישה מוקדמת

Node.js מותקן על המחשב ומפתח API מחשבון Anthropic.

שלב 1: התקנה בטרמינל

npm install -g @anthropic-ai/claude-code

שלב 2: הגדרת API Key

מגדירים את ה-API Key כמשתנה סביבה בטרמינל.

שלב 3: הפעלה

ניגשים לתיקיית הפרויקט ומריצים: claude. מתחיל צ'אט עם גישה לכל הקבצים.

שלב 4: בקשה ראשונה

כותבים לדוגמה "Fix all the TypeScript errors in this project" - הסוכן סורק ומתקן.

⏱️ זמן מוערך: 20-40 דקות כולל הגדרת הסביבה לראשונה.

🧠 Web Chats - כבר יודעים, אבל...

אם כבר משתמשים בצ'אטים - כמה טיפים שישדרגו אתכם מיד:

תנו קונטקסט מלא בהתחלה: "אני בונה אתר לפיזיותרפיסט, קהל יעד מבוגרים. צריך עיצוב פשוט ונקי. בנה HTML+CSS+JS מלא" - לא "תבנה לי אתר".
השתמשו ב-Claude לקוד: Claude מצטיין בדיוק ובעקיבה אחרי הוראות ארוכות - מומלץ לניסיונות ראשונים בקוד.
Gemini לניתוח מסמכים גדולים: חוזה, spreadsheet ענקי, מסמכים ארוכים - Gemini עם חלון הקונטקסט הגדול שלו הוא הבחירה.
GPT לאינטגרציות: אם כבר יש מנוי - הכלים המחוברים כמו Canva ו-GitHub שווים ניסיון.
⏱️ זמן עד תוצאה ראשונה: 2-5 דקות. פשוט מתחילים לכתוב.

🚀 Antigravity - גישה בבטא

חשוב לדעת: הכלי בבטא עם גישה מוגבלת - יתכן שתצטרכו להצטרף לרשימת המתנה.

שלב 1: גישה

זמין דרך Chrome או הורדה ייעודית. מצריך חשבון Google.

שלב 2: יצירת Workspace

פותחים Workspace חדש ומגדירים Knowledge Items - מה שהסוכן יזכור תמיד לגבי הפרויקטים שלכם.

שלב 3: עבודה

הממשק קרוב מאוד ל-Cursor - יש IDE, טרמינל, סוכן וצ'אט. ההבדל המרכזי: שליטה בדפדפן וסוכנים מרובים.

⏱️ זמן מוערך: 10-20 דקות אחרי שיש גישה.

שורה תחתונה

💬 מה אומרים אנשים?

Cursor

משתמשים רבים מתארים אותו כסביבת עבודה נוחה מאוד למי שרוצים לבנות עם AI מתוך ממשק שמרגיש כמו IDE (חלון צ'אט סוכן) אמיתי, אוהבים במיוחד את הזרימה עם הקבצים והסוכן, וחלקם מציינים שמי שלא מגיעים מרקע טכני עדיין עלולים להרגיש הצפה בהתחלה.

Claude Code

מי שאוהבים לעבוד קרוב לקוד ולטרמינל נוטים לתאר אותו ככלי חזק מאוד. אחרים מרגישים שהוא פחות "ידידותי למתחילים" ודורש יותר ביטחון טכני.

Antigravity

מונחים

מונחים ששווה לזכור

סוכן (Agent)

החלק ש"עושה" ולא רק "מדבר". הוא מחליט לבד איזה קובץ לפתוח ואיזו פקודה להריץ.

Skills

מיומנויות שהסוכן לומד (כמו: איך לשלוח אימייל, איך לבדוק דומיין). אפשר להרחיב את הסוכן עם עוד מיומנויות.

Composer

ב-Cursor, זה המקום שבו כותבים בקשה גדולה (כמו: "תבנה לי אפליקציית כושר") והוא בונה את כל הקבצים במקביל.

ריפוזיטורי (Repository)

מחסן קוד - המקום שבו כל קבצי הפרויקט שלכם מאוחסנים ומנוהלים. בדרך כלל ב-GitHub. מאפשר לעקוב אחרי שינויים ולשתף עם אחרים.

אחסון קוד

המקום שבו יושבים קבצי הקוד של הפרויקט (GitHub, GitLab). שונה מאחסון האתר - זה המטבח, לא החלון הראשי.

אחסון אתר (Hosting)

השרת שמריץ את האתר שלכם ומגיש אותו למבקרים. פלטפורמות נפוצות: Vercel, Netlify, Google Cloud. זה מה שגורם לאתר להיות נגיש לכולם.

דומיין

הכתובת של האתר שלכם באינטרנט - לדוגמה yourbusiness.com. קונים אותו בנפרד (Google Domains, GoDaddy ועוד) ומחברים לאחסון האתר.

דף נחיתה

אתר של עמוד יחיד שנועד לפעולה אחת ברורה - למשל השארת פרטים, קנייה, או הרשמה. בניגוד לאתר מלא עם כמה עמודים וניווט.

אתר עם עמודים

אתר עם יותר מעמוד אחד - דף הבית, עמוד אודות, שירותים, צור קשר וכו'. מורכב יותר לבנייה ולניהול מדף נחיתה.

אחסון דאטה-בייס

המקום שבו האתר שומר מידע דינמי - פרטי משתמשים, הזמנות, תוכן שמתעדכן. שונה מאחסון הקוד - זה המידע שהאפליקציה צוברת לאורך זמן.

Deploy / Production

"העלאה לאוויר" - הרגע שהאתר הופך נגיש לכולם באינטרנט, לא רק אצלכם על המסך.

Context Window

כמה מידע המודל יכול "להחזיק בראש" בו-זמנית. חלון גדול - יכול לנתח מסמכים ארוכים יותר בבת אחת.
